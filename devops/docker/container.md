## 컨테이너란?

- 컨테이너는 애플리케이션과 그 실행에 필요한 모든 종속성(라이브러리, 설정, 코드 등)을 하나로 패키징한 실행 단위 이다.
- 이러한 패키징 덕에 어떤 환경에서든 동일하게 동작하도록 보장할 수 있다.


## 컨테이너의 사용이유?

- 개발에서 가장 큰 어려움 중 하나는 환경 설정의 불일치이다.
- 운영체제, 패키지 버전, 환경 변수 등의 차이로 인해 코드가 동일하게 작동하지 않는 문제가 생긴다.
- 컨테이너를 사용하면 개발자마다 다른 환경을 맞출 필요 없이, 같은 이미지로 동일한 실행 환경을 재현할 수 있다.


## 도커란?

- 도커는 컨테이너에 필요한 기능을 거의 대부분 제공하는 플랫폼이다.

## 도커에서 컨테이너가 만들어지는 순서

- **도커 파일** : 패키지, 환경변수설정 등을 기록한 파일 (글만 있는 문서 같은거)
- **도커 이미지** : 컨테이너 실행에 필요한 파일과 설정값, 데이터 등을 포함된 `상태값`이고 하나의 이미지에서 여러개의 컨테이너를 생성할 수 있다. (다만, 컨테이너는 이미지와는 무관)
- **도커 컨테이너** : 컨테이너를 실행시키면 도커 이미지에 설정된 프로그램, 데이터 등이 실제 컴퓨팅 자원에 연결됨
