## 컴파일러(Compiler)

- 소스코드를 한꺼번에 읽어 한번에 기계어로 변환 후 생성된 기계어 코드로 실행 시킨것이다.
- 속도는 컴파일러 과정 자체에선 시간이 걸리지만 변환된 코드는 직접 실행되므로 빠르다.
- 코드를 수정했을땐 다시 컴파일과정이 필요함

**컴파일러에 대표적인 언어로는 C, C++, Go, Rust가 있다.**

---

## 인터프리터(Interpreter)

- 소스코드를 한줄씩 읽어가며 빠른 기계어로 변환하고 실행
- 속도는 컴파일 단계가 없으므로 초기 시작은 빠름 전체 코드 실행시간은 컴파일러를 사용할때보다 느릴 수 있음
- 코드 수정시 컴파일러 과정이 없음

**인터프리터에 대표적인 언어로는 python이 있다.**

---

## JIT 컴파일러

### 예시로 이해해보기

- 컴파일러는 공연 시작 전에 대본 전체를 미리 번역해두고 배우가 그걸 외워서 연기하는 방식
- 인터프리터는 공연 중에 대본을 한 줄씩 읽어 즉석에서 번역하면서 바로 연기하는 방식
- JIT 컴파일러는 처음엔 대본을 즉석에서 읽으며 연기(인터프리터) 하지만, 자주 등장하는 장면은 외워서(기계어로 바꿔서) 나중엔 빠르게 연기하는 방식이다.

**대표적으로 Java(JVM), C#(.NET), node.js(V8)**

### 단계별 정리

| 단계 | 동작내용 | 비유 |
|-----------------|------------------------|----------------------|
| 프로그램 시작 | 코드를 한 줄씩 인터프리터처럼 실행 | 대본을 즉석에서 읽으며 연기 |
| 핫스팟(hot spot) 감지 | 반복문이나 함수 등 자주 실행되는 부분을 모니터링 | 이장면 계속나오네? |
| JIT 컴파일 | 그 부분만 기계어로 변환하고 캐시에 저장 | 이 장면은 외워서 바로 연기하자 |
| 최적화된 실행 | 다음부터는 그 기계어 코드를 바로 실행 | 외원 장면은 빠르게 진행 | 

### 장단점

**장점**

- 실행 중 코드 최적화 가능
- 반복 실행되는 부분은 매우 빠름
- 인터프리터보다 전체 성능 향상

**단점**

- 실행 초반엔 분석/컴파일 때문에 느릴 수 있음
- 메모리 사용량이 늘어남
