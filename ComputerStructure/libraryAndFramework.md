# 라이브러리 vs 프레임워크

📅 작성일: 2025-11-14  
📂 분류: ComputerStructure  
🔖 태그: #Library #Framework

---

## 개요

라이브러리와 프레임워크는 모두 재사용 가능한 코드의 집합이지만, 제어 흐름의 주도권이 어디있는가에 따라 구분된다.

개발자가 코드를 호출하는가? (`라이브러리`) vs `프레임워크`가 개발자의 코드를 호출하는가의 차이이다.

--

## 핵심 개념

| 구분 | 라이브러리 | 프레임워크 |
|-----------|--------------------|---------------------------------|
| 정의 | 특정 기능을 수행하기위한 코드 모음 | 애플리케이션의 뼈대를 제공하는 구조 |
| 주도권(제어 흐름) | 개발자가 라이브러리를 직접 호출 | 프레임워크가 개발자의 코드를 호출(IoC) |
| 제어의 역전 (IoC) | 없음 - 개발자가 제어 | 있음 - 프레임워크가 제어 |
| 구성 요소 | 함수, 클래스 단위의 기능 | 실행 흐름, 생명 주기, 규칙 포함 |
| 유연성 | 자유도가 높고 부분 사용 가능 | 정해진 구조 내에서 사용해야 함 | 

---

## 작동 방식 예시

- 라이브러리 사용 예시 (Javascript)

```javascript
import _ from "lodsh";

const arr = [1, 2, 3];
const doubled = _.map(arr, (n) => n * 2);  // 개발자가 직접 호출
```

- 프레임워크 사용 예시 (SpringBoot)

```java
// 프레임워크(Spring)가 @GetMapping을 해석하고 요청시 메서드 호출
@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello World";
    }
}
```

## 핵심

- 라이브러리는 "내가 가져다 쓰는 도구"
- 프레임워크는 "내가 그 안에서 움직이는 틀"
- 핵심 차이는 제어의 역전(IoC)
- 프레임워크는 개발자가 정한 코드의 흐름을 제어하지 않고 정해진 생명주기 안에서 동작한다.
- 라이브러리는 호출과 사용의 자유도가 높지만, 프레임워크는 일관된 구조와 규칙 속에서 개발 효율과 유지보수를 높인다.
